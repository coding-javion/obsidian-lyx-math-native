# LyX Math Native for Obsidian

[中文说明](README.zh-CN.md)

LyX Math Native is a desktop-only Obsidian plugin that edits formulas through a
native C++ sidecar linked against LyX's math editor code. The goal is not to
reimplement LyX math input in JavaScript. Obsidian provides the plugin shell,
while `lyx-mathd` owns the formula tree, LyX-style structured editing, cursor
movement, layout metrics, and native Qt painter rendering.

This project is independent from JavaScript-only LyX-style math plugins.

## Status

This is an early native prototype. The current packaged release target is:

- macOS arm64
- Obsidian desktop
- LyX 2.5.1 math code
- Qt app-bundle sidecar packaged with `macdeployqt`

Mobile Obsidian is not supported because the plugin launches a local native
process.

## Features

- Opens a small LyX-style formula editor from Obsidian commands.
- Inserts inline formulas and display formulas with separate commands.
- Uses LyX `mathed` structures for parsed formula state instead of treating the
  editor as a plain LaTeX preview.
- Renders through LyX/Qt painter output at high resolution.
- Supports common structured math input, including fractions, Greek macros,
  matrices, cases, multiline display formulas, cell navigation, row insertion,
  and column insertion.
- Preserves matrix structure when switching between inline and display mode.
- Uses a bundled LyX support directory, so users do not need to install LyX just
  to run the plugin package.

## Installation

End users should install a release zip. They should not build LyX or clone this
repository into an Obsidian vault.

Download the release zip for your platform from GitHub Releases. For the current
macOS arm64 build, the asset name is similar to:

```text
lyx-math-0.0.1-darwin-arm64.zip
```

Install it manually:

1. Open your vault's plugin directory:

   ```text
   <vault>/.obsidian/plugins/
   ```

2. Create this directory:

   ```text
   lyx-math
   ```

3. Extract the zip contents into that directory. The directory should contain
   `manifest.json`, `main.js`, `styles.css`, `bin/`, and `support/`.

4. Restart Obsidian or reload plugins.
5. Enable `LyX Math Native` in Community plugins.

## Commands

The plugin registers these Obsidian commands:

- `Insert inline LyX formula`
- `Insert display LyX formula`
- `Edit native LyX formula at cursor`
- `Insert matrix row in LyX formula editor`
- `Insert matrix column in LyX formula editor`
- `LyX native sidecar status`

Inline and display formulas are intentionally inserted through different
commands because the sidecar stores the corresponding LyX hull style.
The matrix row and column commands are intended for Obsidian's Hotkeys
settings while the LyX formula editor modal is open.

## Repository Layout

- `manifest.json`, `main.js`, `styles.css`: Obsidian plugin runtime.
- `src/native_client.js`: Node client for the native sidecar.
- `native/lyx-mathd`: C++ sidecar source.
- `scripts`: build, packaging, and release helpers.
- `patches`: patch needed to expose the LyX 2.5.1 math core as a sidecar target.
- `support/lyx`: packaged LyX runtime resources generated by
  `npm run support:package`.
- `bin/<platform>-<arch>`: packaged native sidecar generated by native packaging
  scripts.
- `dist`: generated installable release zips.

## Build From Source

This section is for developers and release maintainers. Normal users should use
the release zip instead. See [BUILDING.md](BUILDING.md) for the full build notes.

The short macOS development flow is:

```bash
cd /path/to/ObsidianLyX
export LYX_SOURCE_DIR=/path/to/lyx-2.5.1
export LYX_BUILD_DIR=/tmp/lyx-2.5.1-qt-build

cd "$LYX_SOURCE_DIR"
patch -p1 < /path/to/ObsidianLyX/patches/lyx-2.5.1-mathd-cmake.patch

cd /path/to/ObsidianLyX
npm run lyx:configure
npm run lyx:build-core
npm run native:configure:lyx
npm run native:build
npm run native:package
npm run support:package
npm test
```

For a self-contained macOS release sidecar:

```bash
npm run native:package:macos-app
npm test
npm run release:all
```

The generated release assets are written to `dist/`.

## Testing

Run:

```bash
npm test
```

The test starts the packaged sidecar, checks the native protocol, verifies
LyX-backed parsing and rendering, exercises macro completion, deletion, cursor
movement, matrix editing, display switching, multiline formulas, and basic
Obsidian math range scanning.

## Troubleshooting

If Obsidian reports that `lyx-mathd` cannot be spawned, check that:

- the installed plugin directory contains `bin/<platform>-<arch>/`;
- the release zip matches your CPU architecture;
- `lyx-mathd` is executable;
- macOS has not quarantined the unpacked files.

If the plugin loads but the sidecar status command reports unavailable, run the
test from the plugin directory:

```bash
npm test
```

For local unsigned macOS builds, Gatekeeper may block copied binaries. Public
releases should be signed with Developer ID and notarized.

## Release Process

The repository includes a GitHub Actions workflow at
`.github/workflows/release.yml`.

- Manual `workflow_dispatch` builds upload workflow artifacts.
- Pushing a tag like `0.0.1` builds the macOS arm64 release and publishes a
  GitHub Release.
- Each release should contain `main.js`, `manifest.json`, `styles.css`, the
  installable plugin zip, and the corresponding source tarball.

For a local macOS arm64 release:

```bash
npm run native:build
npm run native:package
npm run support:package
npm run native:package:macos-app
npm test
npm run release:all
```

Upload the generated files under `dist/` as GitHub Release assets:

```text
main.js
manifest.json
styles.css
lyx-math-0.0.1-darwin-arm64.zip
lyx-math-0.0.1-source.tar.gz
```

Users install the zip contents into their vault. The source tarball is for GPL
corresponding source and developer rebuilds.

See [RELEASE_CHECKLIST.md](RELEASE_CHECKLIST.md) before publishing.
